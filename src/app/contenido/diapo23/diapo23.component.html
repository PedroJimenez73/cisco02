The network administrator cannot connect to Switch 1 over a Telnet session, although the hosts attached to Switch1 can ping the interface Fa0/0 of the router. Given the information in the graphic and if the router and Switch2 are configured properly, which of the following commands should be issued on Switch1 to correct this problem?

Solution:

Configure the command ip default-gateway with the address 192.168.24.1 in switch1.

Without the default-gateway configured on a host on a network, it cannot return packets outside its network and in this case, as the administrator who wants to connect this out of the network, we have no choice but to configure the default-gateway.
